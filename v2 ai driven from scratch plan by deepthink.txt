### PART 1: Technical Analysis & The "AI-Native" Kernel Architecture

The optimal architecture for an AI-generated NWP system is a **Python-Orchestrated, JIT-Compiled JAX/Triton Hybrid**.

#### 1. Language & Ecosystem

* **The Orchestrator:** `Python 3.12+`. Python handles MPI orchestration (via `mpi4py` and GPU-aware NCCL), asynchronous I/O, and ML integration. It is the native language of AI models, making it trivial for agents to modify the top-level execution graph.
* **The Dynamical Core (Dycore):** `Google JAX`. The Navier-Stokes equations, 3D stencils, and Runge-Kutta 3 (RK3) integration are written in JAX. JAX provides pure functional semantics, immutable arrays, and compiles directly to highly optimized PTX/ROCm via XLA. XLA automatically handles "vertical kernel fusion."
* **The Extreme Physics Layer:** `OpenAI Triton` (or `CuPy` Raw JIT CUDA). Legacy WRF hits a 255-register limit in deep column physics (like MYNN or Thompson), causing "spilling" and destroying performance. The AI will write these dense modules in Triton. Triton forces the AI to explicitly manage block-level memory (SRAM / L1 Shared Memory), directly solving the register pressure crisis without fighting C++ framework bloat.

#### 2. Kernel Execution Paradigm

* **Absolute Device Residency:** The 3D atmospheric state ($u, v, w, t, p, q$) is allocated on the GPU High-Bandwidth Memory (HBM) at $T=0$. It never returns to the CPU except for batched, asynchronous output.
* **Explicit Shared Memory Tiling:** When translating legacy Fortran loops, the AI is prompted to mathematically tile the 3D grid and push intermediate prognostic variables into SRAM block-pointers. Threads will stride through SRAM instead of using local registers, artificially maintaining $>50\%$ GPU occupancy.
* **Mixed Precision:** FP64 is used *strictly* for the dycore's mass continuity and pressure gradients. Chaotic physics tendencies are calculated natively in FP32 or BF16 (utilizing Tensor Cores).

### PART 3: The Verification Pyramid (Test-Driven AI)

We formally abandon full bitwise reproducibility. Before *any* NWP simulation code is written, Agent $\beta$ must build this 4-tier automated testing pipeline:

* **Tier 1: Serialbox Parity (Micro-level):** Legacy CPU WRF is run *once*. Millions of input/output arrays for every single Fortran subroutine are dumped to disk as binary files. Agent $\alpha$'s isolated GPU kernels must match these outputs within a defined FP32 $\epsilon$-tolerance.
* **Tier 2: Conservation Guards (Physics-level):** Hardcoded checks for Mass Conservation, Tracer Positivity (moisture cannot be negative), and Kinetic Energy spectra ($k^{-5/3}$ cascade check).
* **Tier 3: Timestep Convergence (TSC 1.0):** Evaluates if the numerical drift of the new GPU kernel over 10 timesteps is smaller than the natural chaotic sensitivity of the equations themselves.
* **Tier 4: PyCECT (Statistical Macro-level):** A 100-member CPU ensemble with $O(10^{-14})$ perturbations defines the "truth variance" via Principal Component Analysis (PCA). The assembled GPU model is deemed physically correct only if its PCA scores fall within 2 standard deviations of the CPU ensemble.

#### Phase 3: The Physics "Mega-Kernel" Factory (Weeks 11-16)
