CuPy RawKernel has the lightest Python-side setup of the compiled candidates: a sprint-local venv plus `cupy-cuda13x==14.0.1`; no project dependency was added. The cost is that CUDA errors arrive through NVRTC/CuPy layers, so the first useful line is often in a generated compile log rather than at an IDE diagnostic site. I captured a deliberate broken kernel in `data/profiler_artifacts/cupy_or_numba/deliberate_kernel_bug.txt`; it reports an NVRTC compile exception with source line context, which is usable but noisier than `nvcc`.

The debugger story is mixed. Nsight Compute can wrap the Python module, and RawKernels can be profiled as CUDA kernels, but local performance-counter permission still limits metric collection. `cuda-gdb` can attach to the Python process, though iteration is slower than a native binary. Agent friction was low for algorithm changes because host orchestration stayed in Python; the fragile parts were RawKernel attribute names and occupancy access across CuPy versions.
