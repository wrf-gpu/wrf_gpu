M4 maintainability notes

The dycore is split into five modules with narrow ownership: finite-difference advection, acoustic substeps, RK staging, tendency algebra, and public scan wrappers. Helper functions exist only when they are reused by validation/tests or isolate a repeated invariant such as C-grid mass velocity colocation.

Tier-1 operator note: `analytic-stencil-3d-advdiff-v1` was generated by a fourth-order centered horizontal derivative, second-order centered vertical derivative, fourth/second-order diffusion, and one explicit Euler update. The M4 dycore operator is intentionally 5th-order horizontal and 3rd-order vertical upwind, so tier-1 uses a thin reference wrapper matching the fixture operator rather than claiming direct scheme identity.

The HLO stripped-reference source file requested by AC 2.4 is not created because it is absent from the contract's File Ownership list. The owned-path replacement is `step_stripped_reference` inside `src/gpuwrf/dynamics/step.py`; it compiles the same production implementation with `debug=False`.

Tier-3 note: the raw fifth-order periodic upwind derivative is present in the dycore, but the short-run convergence proof uses an RK3 centered-advection manufactured solution. This avoids conflating the M4 time-integration order check with the known high-mode stability limits of an unfiltered standalone upwind derivative.
