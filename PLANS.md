# Execution Plan Template

Copy this into `codex/plans/` or a sprint folder before implementation.

## Objective

What this plan will prove or deliver.

## Non-Goals

What is intentionally excluded.

## Prerequisites

Inputs, fixtures, hardware, tools, and decisions required before work starts.

## Files Allowed To Change

Exact paths or ownership boundaries.

## Acceptance Criteria

Observable pass/fail criteria.

## Validation Commands

Commands that must run before handoff.

## Performance Metrics

Metrics, profiler commands, JSON fields, and acceptance thresholds.

## Risks

Known ways this plan can fail.

## Rollback

How to revert or disable the work.

## Review Requirements

Required reviewer, tester, ADR, or human approval.
