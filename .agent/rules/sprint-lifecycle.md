# Sprint Lifecycle

1. Manager creates sprint folder from template.
2. Manager writes sprint contract: objective, scope, file ownership, acceptance criteria, validation commands, and proof object.
3. Worker implements only the contracted work.
4. Tester runs independent checks or creates missing tests.
5. Reviewer challenges correctness, performance, and scope.
6. Manager closes with decision, artifacts, risks, and memory-patch proposal.

A sprint without a proof object stays open.
