## Implementation Summary

Added separate JAX SW and LW kernels. SW state carries `T`, `p`, `qv`, `qc`, `qi`, `qs`, `qg`, `cloud_fraction`, `surface_albedo`, `coszen`, `dz`, and `rho` (`src/gpuwrf/physics/rrtmg_sw.py:32`), and SW outputs include heating rate, bottom-to-top interface fluxes, TOA/surface diagnostics, atmospheric absorption, and surface absorption (`src/gpuwrf/physics/rrtmg_sw.py:89`). LW state carries the same thermodynamic/hydrometeor/layer fields plus `surface_temperature` and `surface_emissivity` (`src/gpuwrf/physics/rrtmg_lw.py:31`), and LW outputs include heating rate, interface fluxes, TOA/surface diagnostics, column net heating, and surface emission (`src/gpuwrf/physics/rrtmg_lw.py:88`).

The table loader exposes `RRTMGTableBundle` as JAX array leaves (`src/gpuwrf/physics/rrtmg_tables.py:34`) and loads arrays from `data/fixtures/rrtmg-tables-v1.npz` as `jnp.asarray` leaves (`src/gpuwrf/physics/rrtmg_tables.py:66`). The kernels retrieve spectral coefficients with `jnp.take` in the traced body (`src/gpuwrf/physics/rrtmg_sw.py:143`, `src/gpuwrf/physics/rrtmg_lw.py:135`) and pass the table bundle through the jitted entry points (`src/gpuwrf/physics/rrtmg_sw.py:191`, `src/gpuwrf/physics/rrtmg_lw.py:191`).

Important source-discovery correction: the sprint text said "32 bands", but the local WRF source declares 14 shortwave bands (`nbndsw = 14`) and 16 longwave bands (`nbndlw = 16`), so this implementation follows the discovered WRF build rather than padding SW to 16 bands (`/mnt/data/canairy_meteo/artifacts/wrf_gpu_src/WRF/phys/module_ra_rrtmg_sw.F:29`, `/mnt/data/canairy_meteo/artifacts/wrf_gpu_src/WRF/phys/module_ra_rrtmg_sw.F:31`, `/mnt/data/canairy_meteo/artifacts/wrf_gpu_src/WRF/phys/module_ra_rrtmg_lw.F:76`, `/mnt/data/canairy_meteo/artifacts/wrf_gpu_src/WRF/phys/module_ra_rrtmg_lw.F:78`). ADR-009 records this explicitly (`.agent/decisions/ADR-009-rrtmg-jax-implementation.md:16`).
