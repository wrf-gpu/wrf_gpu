# M5-S0 First-Physics-Suite Scout Report

Summary: This scout recommends Thompson microphysics as the first M5 physics scheme, using WRF v4.7.1 Thompson 2008 `mp_physics=8` semantics as the target behavior for M5-S1. The recommendation is not because microphysics is the only operationally important Canary scheme. Marine PBL treatment and radiation are also critical. Thompson is the best first scheme because it has immediate value for 3 km shower and terrain precipitation forecasts, can be isolated as a column source-term fixture, exercises the real M5 register-pressure risk, and does not require the surface/SST/static-geog coupling that the roadmap defers.

Decision: Pick Thompson microphysics first. Treat M5-S1 as a stop/go test for JAX on real WRF column physics under ADR-001: proceed in pure `jax.jit` plus `jax.numpy` plus `jax.lax` if profile evidence stays within gate; open the per-scheme Triton fallback mini-ADR only if the real Thompson profile trips the gate.

## Candidate Inventory

| Category | Paragraph |
|---|---|
| Microphysics | Microphysics computes phase changes, hydrometeor production/loss, fallout/sedimentation, latent heating, and precipitation formation. WRF deployments commonly use Thompson, WSM6, Morrison-Gettelman, or P3 because these schemes decide whether explicit 3 km precipitation becomes rain, graupel, snow, or evaporating virga. Thompson is widely used because it is more physically expressive than simple warm-rain schemes while remaining an operational bulk scheme. Primary cost is branching and register pressure: many saturation, freezing, collection, melting, and sedimentation pathways per vertical level, plus memory traffic across hydrometeor arrays. |
| Planetary Boundary Layer / surface layer | PBL and surface-layer schemes compute turbulent vertical mixing, entrainment, near-surface exchange, and often TKE or nonlocal transport. WRF deployments use MYNN2.5/MYNN-EDMF, YSU, MYJ, or ACM2 because near-surface wind, temperature, humidity, cloud base, and inversion structure are PBL controlled. For Canary trade winds this category is operationally essential. Primary cost is branching plus vertical-column coupling: stability functions, mixing-length logic, surface-flux dependencies, and implicit or semi-implicit vertical diffusion solves can create register and local-memory pressure. |
| Radiation | Radiation computes shortwave and longwave heating, cloud-radiative interaction, surface radiative fluxes, and the diurnal energy cycle. RRTMG SW+LW is common in operational WRF stacks because it is accurate enough for weather timescales and interfaces with cloud and gas profiles. Goddard and CAM are alternatives. Primary cost is arithmetic intensity, lookup-table access, and many spectral-band loops. It is less branch-chaotic than microphysics but much larger and more table/input dependent. |
| Land surface | Land-surface schemes compute soil temperature/moisture, runoff, vegetation, snowpack, skin temperature, and land-atmosphere fluxes. Noah, Noah-MP, and RUC are common because surface fluxes and soil memory strongly affect 2 m temperature, humidity, and boundary-layer evolution. Primary cost is moderate branching and memory across soil/snow/vegetation state, with strong dependence on static geog fields and surface datasets. For M5 it is disadvantaged because surface/land/SST/static-geog proof objects are deferred unless the selected first scheme requires them. |
| Convection | Cumulus schemes parameterize unresolved deep or shallow convection, often including mass flux, entrainment/detrainment, and convective precipitation. Kain-Fritsch, Grell-Freitas, Tiedtke, and shallow-convection options are common in coarser WRF nests. At 3 km this is gray-zone: explicit convection is often viable for deep convection, but shallow-cumulus or outer-domain cumulus treatment may still matter. Primary cost is branching and column integration. It is not a good first M5 target because v0 is one 3 km domain path and explicit convection avoids an early cumulus commitment. |

## Canary-Specific Operational Fit

1. Planetary Boundary Layer / surface layer: highest meteorological criticality. Canary forecasts are dominated by marine trade winds, stable inversion-capped boundary layers, cloud-top entrainment, and terrain acceleration. PBL choice strongly affects 10 m wind, 2 m humidity/temperature, cloud depth, and low-cloud persistence. It is not the best first implementation because MYNN-style schemes depend on surface-layer fluxes and vertical mixing solves that make validation and JAX profiling harder before M5 has any real physics proof.

2. Microphysics: highest first-implementation fit. Terrain-induced precipitation on windward slopes and frequent shallow convective showers at 3 km need explicit hydrometeor and latent-heating behavior. Microphysics can be isolated in a WRF column fixture, has clear Tier 1 parity variables and Tier 2 water/positivity invariants, and does not require a new SST/static-geog ingestion proof object. It also directly tests the M5 risk ADR-001 named: real WRF branchy column physics under JAX/XLA.

3. Radiation: important for day-to-day heating and cloud-radiative feedback, especially daily cycles over islands, but less urgent than hydrometeor source terms for the first column proof. RRTMG introduces spectral tables, gas/aerosol assumptions, solar geometry, and time-of-day coupling. That makes it a poor first stop/go test even though it must arrive soon for useful daily forecasts.

4. Land surface: important over island interiors, slopes, and land-sea contrast, but M5 has no SST coupling requirement and M7 is where operational surface/static-geog proof objects become unavoidable. Noah-MP first would expand inputs and state before the physics kernel path is proven.

5. Convection: deferred. At 3 km, explicit deep convection plus microphysics is a defensible first path. Shallow convection may need attention for trade-wind cumulus, but a cumulus parameterization first would conflict with the v0 preference for one domain path and would be less directly fixture-backed.

## Recommendation

Pick **Thompson 2008 microphysics with WRF v4.7.1 `mp_physics=8` semantics** as the first M5 scheme. It primarily improves explicit precipitation, hydrometeor phase, latent heating, and terrain/shower precipitation realism for the Canary 3 km daily target. It is operationally important without forcing M5 to solve surface/SST/static-geog coupling early. JAX implementability is risky but suitable for the first real physics stop/go: the scheme can be staged into a small number of column kernels and should be attempted in pure JAX before invoking ADR-001's Triton fallback. Bakeoff-validation tractability is strong: WRF v4.7.1 can generate isolated column savepoints for Thompson source terms, while Tier 2 checks are natural water budget closure, tracer positivity, bounded tendencies, and NaN/Inf absence. The M2 analytic column surrogate does not prove success, but it gives a clean starting point and the M5 gate is specifically designed to decide this.

## Out-of-Scope Notes

MYNN2.5/MYNN-EDMF PBL is deferred to M5-S2 or later, not because it is unimportant, but because it couples tightly to surface fluxes and vertical diffusion structure. RRTMG SW+LW is deferred until after the first column physics path is validated; it needs solar geometry, gas/ozone/cloud optical inputs, and table governance. Noah-MP is deferred until the surface/land/SST/static-geog proof object is scheduled. Cumulus parameterization is deferred because the 3 km v0 path should first exercise explicit convection/microphysics, with shallow-convection need revisited after coupled short runs.

## Risks + Open Questions

- Thompson may exceed the register/local-memory gate when translated faithfully from WRF. If so, this is not a sprint failure; it triggers ADR-001's per-scheme Triton fallback memo with actual profile evidence.
- The exact WRF v4.7.1 Thompson savepoint boundary must be selected before M5-S1: call-site inputs, outputs, and whether sedimentation is included in the first parity fixture need manager/reviewer approval.
- A domain expert should later confirm whether shallow-cumulus treatment is needed for the Canary 3 km operational domain, especially for trade-wind cumulus under inversion.
- Radiation and PBL remain operationally critical. Deferring them is a sequencing decision, not a claim that precipitation physics alone makes a useful daily forecast.
- M5-S1 should record a tester waiver only if the manager explicitly keeps the research-sprint waiver; implementation sprints need tester evidence.
