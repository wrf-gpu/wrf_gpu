Per-line commentary
-------------------

1. `_acoustic_scan` signature
   - BEFORE: substep count was hard-coded to `namelist.acoustic_substeps`.
   - AFTER:  added kw-only `substeps: int | None = None` override.
   - dt_sub denominator now matches the override (`dt_stage / scan_substeps`),
     so a single-substep call uses the full stage dt (correct for WRF RK1).
   - jax.lax.scan length now uses `scan_substeps`, not the namelist default.
   - VERDICT: substantive plumbing change, not a no-op.

2. `advance_stage` signature
   - BEFORE: bool `use_acoustic` toggled the acoustic call on/off.
   - AFTER:  `acoustic_substeps: int | None`. `None` skips, otherwise scans.
   - The conditional is now `if acoustic_substeps is not None`, which for the
     RK1 case (substeps=1) IS truthy and so calls `_acoustic_scan`.
   - VERDICT: no `if rk_step == 1: skip_acoustic` guard remains.

3. RK1/RK2/RK3 dispatch (jax.lax.switch branches)
   - BEFORE: RK1 -> use_acoustic=False; RK2 -> True; RK3 -> True.
   - AFTER:  RK1 -> substeps=1; RK2 -> substeps=namelist.acoustic_substeps;
             RK3 -> substeps=namelist.acoustic_substeps.
   - Comment cites WRF anchor: `solve_em.F:1472-1475` "number_of_small_timesteps = 1"
     for RK3 stage 1 (which our code calls RK1 since rk_order==3).
   - VERDICT: RK1 acoustic call is now *unconditionally* wired with substeps=1.
     The argument is a Python int constant so `if acoustic_substeps is not None`
     always evaluates True at trace time.

4. Cosmetic check (anti-no-op)
   - No `pass` statement was added.
   - No no-op wrapper (e.g. `_acoustic_scan(..., substeps=0)` would be no-op;
     substeps=1 is one real WRF small step).
   - No hidden conditional in `_acoustic_scan` body that would short-circuit
     when substeps=1. The body unconditionally runs `_wrf_small_step_acoustic`
     when `use_vertical_solver=True` (the operational default).

5. Subsequent commit 268e38d (D2H lift) modifies this same region
   - Replaces `jax.lax.switch` with three sequential `advance_stage` calls.
   - RK1 still receives `acoustic_substeps=1`, the `if .. is not None` branch
     is removed entirely (always calls `_acoustic_scan`).
   - The semantic change is identical: RK1 runs exactly one acoustic substep
     at the operational `dt_rk = dt/3` cadence.

Static conclusion for Part 1
----------------------------
The diff at commit 879ef56 wires an *actual* acoustic substep at RK1 by
plumbing a `substeps` override through `_acoustic_scan` and dispatching the
RK1 branch with `substeps=1`. The control flow now enters
`_wrf_small_step_acoustic` (when `use_vertical_solver=True`) once per RK1
stage. This is structurally consistent with WRF `solve_em.F:1472-1475`.

The diff is NOT cosmetic. It is, however, narrower than the validation
acoustic loop: see Part 3 for the operational-vs-validation semantic gap.
