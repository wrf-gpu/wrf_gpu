# WRF GPU Port Catalogue

Scope: WRF-only GPU/offload efforts. Non-WRF GPU NWP systems such as COSMO,
ICON, SCREAM, NIM, Pace/FV3, MPAS, and AI weather models are important context
but are deliberately excluded from the table.

| FahrenheitResearch `wrf-gpu-port` | 2026 | Independent GitHub repository | Python patch scripts injecting NVHPC OpenACC into WRF 4.7.1 generated `.f90` | Dynamical core small_step, big_step, diffusion, module_em, solve_em; physics, boundary conditions, I/O, and advection remain CPU/disabled per README | README claims 6x on a 3 km test and successful 250 m run; also admits much observed speedup may come from NVHPC compiler optimizations | MIT license | Active partial open-source directive port in 2026; not full physics/device-resident WRF [sources: fahrenheit2026wrfgpuport,ucar2026forumFahrenheit] |

Catalogue takeaway: the strongest defensible negative claim is not "no GPU WRF
exists." It is: no prior public evidence found in this sprint for a fully
source-open, full-dynamics-plus-operational-physics, high-frequency
device-resident WRF v4-compatible port with open validation artifacts.
