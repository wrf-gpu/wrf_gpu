### C-primary

`C-primary = ICON4Py-pattern JAX rewrite`, not direct ICON4Py dependency.

Reason: ICON4Py has the strongest Option C numerical substrate evidence for this particular failure class: regional/limited-area hooks, named nonhydrostatic configuration, explicit vertical implicit predictor/corrector programs, tridiagonal `w` solve, Rayleigh/divergence damping structure, and field exchanges (`solve_nonhydro.py:139-194,307-352,543-621,1195-1232,1333-1378`; `vertically_implicit_dycore_solver.py:139-206,283-356`). It is stronger than Dinosaur/NeuralGCM for WRF compatibility because those are global spectral/sigma systems (`Dinosaur README.md:5-10`; NeuralGCM public `api.py:413-416,425-447`). It is stronger than Pace as a regional pattern, though Pace remains the best split-acoustic architecture reference (`dyn_core.py:701-720,793-805,874-892`).

The direct dependency path is not recommended: ICON4Py is GT4Py, experimental, and not packaged on PyPI (`README.md:7-19`), while ADR-001 selected JAX primary and records GT4Py exclusion due tooling (`.agent/decisions/ADR-001-backend-selection.md:11-22`). The viable C version is therefore a port of the ICON4Py nonhydro pattern into this repository's JAX `State`/Gen2 replay contracts.

## Scout Recommendation

Pure Option C is not the best immediate M6 close path. It is an important fallback design direction, and C-primary should be the ICON4Py-pattern JAX rewrite, but every direct substrate creates a state/grid/validation adapter before the project can even run the existing d02 replay. Between A and B, B has the best chance of producing an honest 1h coupled forecast because it uses WRF savepoints to remove ambiguity from each small-step recurrence. A is faster, but it risks spending another sprint series on a partial scratch set that still misses `muts`, `ph_tend`, `_1`, or `_save` interactions visible in WRF `module_small_step_em.F:1102-1489`.
