interface and gate the column implementation against real WRF-module
savepoints for unstable, stable, neutral, and edge-case surface-forcing regimes.

## Unresolved Risks

- The oracle is a real WRF-module oracle, not a full coupled `wrf.exe` real-case
  savepoint. Integrated realism remains a later mixed-suite gate.
- The implementation is column-level and parity-gated for the no-BEP/no-chemistry
  YSU path represented by the oracle. Domain batching and operational dispatcher
  wiring remain manager/integrator work.
- Top-down cloud-driven YSU mixing is not claimed here because the generated
  YSU oracle cases set `ysu_topdown_pblmix=0` and `qc/qi=0`.
- No GPU performance claim is made.
