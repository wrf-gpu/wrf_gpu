# GPT v0.4.0 V10 First-Hour Spin-Up Diagnosis

## Objective

Diagnose the remaining v0.4.0 native-init forecast V10 residual and decide whether it is a fixable WRF-faithful defect, acceptable first-hour spin-up, or council-worthy ambiguity.

## Verdict

**NEEDS-COUNCIL.**

The evidence rules out finer LBC cadence as an h1 explanation and confirms a real MYNN first-call QKE initialization gap. I implemented a conservative WRF-style cold-start QKE seed and ran the single authorized GPU smoke. It did **not** close the gate:

- `proofs/v040/s5_forecast_gate_report.json`: `foundation_confirmed=false`, verdict `STABLE_BUT_CORE_FIELD_MISMATCH`.
- h1 V10 bias after patch: `+0.5171465726504967 m/s`, margin `0.2752320537920854`.
- h2 V10 bias after patch: `+0.20756419184470878 m/s`, within margin.
- Forecast remains stable/physical for 6h.

This is not cleanly (A), because the implemented WRF-style seed was not sufficient. It is not cleanly (B), because native MYNN first-call behavior was demonstrably not WRF-faithful before the patch, and h1 remains a real verified miss after the patch.

## Diagnosis

### Suspect A: missing spun-up turbulence/surface state at t0

Native IC is **not missing a nonzero WRF wrfinput QKE field**:

- Native t0 `State.qke`: max `0.0`, mean `0.0`, nonzero count `0`.
- WRF `wrfinput_d01:QKE`: max `0.0`, mean `0.0`, nonzero count `0`.
- Native minus WRF wrfinput QKE: max abs `0.0`.
- WRF `wrfinput_d01:qke_adv`: present and zero.

WRF t0 wrfout surface/turbulence diagnostics are also effectively unspun:

- WRF `wrfout_t0:QKE`: zero.
- WRF `wrfout_t0:TKE_PBL`: zero.
- WRF `wrfout_t0:PBLH`: zero.
- WRF `wrfout_t0:UST`: constant `~1.0e-4`.

Native first surface-layer call computes real nonzero friction velocity:

- Native `surface_adapter` t0 `ustar`: min `0.0128356531`, max `0.347543029`, mean `0.135819696`.

Conclusion: real.exe/wrfinput does not carry a spun-up QKE reservoir for this cold-start case. The relevant difference is the **first MYNN call**, not a missing wrfinput QKE load.

### Suspect B: 6h to 1h LBC linearization

Ruled out for h1 in this smoke:

- `lbc_intervals=1` product has one 6h tendency interval.
- `lbc_intervals=2` product has two 6h tendency intervals.
- Forecast boundary leaf h1 values are identical for all state leaves: max abs diff set is `{0.0}`.
- Source reason: `build_native_boundary_leaves()` consumes interval `0` via `v[0]` and `t[0]` for h1; adding a second 6h interval only supplies later tendency frames, not sub-6h boundary data.

There is no finer-than-6h boundary frame in this corpus to test an actual higher-cadence LBC.

### Suspect C: native qke IC vs WRF first MYNN call

Confirmed as a real discrepancy:

- Native raw first-MYNN-call QKE before patch: max `0.0`, mean `0.0`, nonzero count `0`.
- WRF source enters a first-call initialization branch for non-restart starts and low-QKE restarts. Anchors are recorded in `proofs/v040/v10_spinup_diagnosis.json`:
  - `module_bl_mynnedmf.F90`: `initflag > 0 .and. .not.restart`
  - low-QKE threshold `maxval(qke1) < 0.0002`
  - surface-friction-velocity QKE taper using `ust`
  - `CALL mym_initialize(...)`
  - bottom QKE seed `1.5*ust*ust`
  - minimum `MAX(ust, 0.02)`

Patch implemented:

- Added `_cold_start_qke()` in `src/gpuwrf/physics/mynn_pbl.py`.
- Applied it before MYNN turbulence/prediction, EDMF input, and budget diagnostics when column max QKE `< 2e-4`.
- Candidate seeded first-MYNN QKE: max `1.507472038`, mean `0.013056609`, nonzero count `241428`.

Post-smoke turbulence comparison:

- h1 QKE mean: CPU `0.05797278`, GPU `0.05188116`, bias `-0.00609162`, RMSE `0.1775307`.
- h1 UST mean: CPU `0.16786553`, GPU `0.18348975`, bias `+0.01562422`, RMSE `0.0632392`.
- h2 QKE mean: CPU `0.04826566`, GPU `0.05967630`, bias `+0.01141064`, RMSE `0.2629476`.
- h2 UST mean: CPU `0.14821925`, GPU `0.18178702`, bias `+0.03356777`, RMSE `0.0943594`.

Interpretation: the QKE first-call gap is real, but the conservative seed only slightly reduced h1 V10 (`+0.528` prior to `+0.517` after patch) and did not close the gate.
