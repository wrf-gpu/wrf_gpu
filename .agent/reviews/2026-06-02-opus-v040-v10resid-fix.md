## root cause + fix

### Bug 1 — placeholder surface land state (over-rough ocean)
`build_native_forecast_case` hardcoded `roughness_m=0.1` (the LAND default),
`mavail=1.0`, `lakemask=0` everywhere. The d01 domain is ~90% water; 0.1 is **35x too
rough** vs the WRF ocean seed 2.85e-3. Effect: (a) over-reduces the 10m log-wind
diagnostic; (b) over-drags the prognostic lowest-level wind via the surface-layer
momentum stress `tau = -ustar^2 * u/wspd` every step, so V10 lost ~+0.59 m/s at t0 and
amplified during integration.

Fix: derive `roughness_m`/`mavail`/`lakemask` per-cell from the native
`xland`/`landmask`/`vegfra`/`smois` using the SAME validated logic the d02/d03 replay
path uses — `gpuwrf.physics.noah_mp.roughness_from_prescribed_fields` and
`jnp.where(water, 1.0, top_soil)` (= `load_prescribed_land_state` ->
`prescribe_noah_mp_state`). WRF's `real.exe` does NOT write ZNT to wrfinput; ZNT is
set at the first physics step from the land-use mask (LANDUSE.TBL) / Charnock over
water — exactly what `roughness_from_prescribed_fields` models. CPU check: water
roughness 0.1 → 2.85e-3; t0 V10 bias +0.59 → +0.31.

### Bug 2 — LBC tendency time-level spacing (6x over-application)
The met_em forcing is **6-hourly** (`bdyfrq = 21600 s`), but the operational
`interpolate_boundary_leaf` indexes the leaf time axis at
`lead_seconds / config.update_cadence_s` (`update_cadence_s = 3600 s = 1 h`).
`build_native_boundary_leaves` built the time levels spaced by `bdyfrq`
(`value + k*interval*tend`, k*6 h), so at lead 1 h the applier read leaf level 1 = the
FULL 6-h boundary change and applied it in the first hour — a **6x over-application** of
the boundary tendency, pushing the meridional boundary wind ~+1.5 m/s/hr northward.

Fix: space the leaf time levels by the **applier cadence** `cadence_s`
(= `update_cadence_s`), not `bdyfrq`; threaded `boundary_cadence_s=3600.0` from one
constant into both `build_native_boundary_leaves(cadence_s=...)` and the
`BoundaryConfig(update_cadence_s=...)` so they can never diverge. The native
per-second tendency is unchanged; only the per-level dt is corrected. CPU check:
native 1-h boundary dV (W/E/S/N) 0.48/0.48/0.75/1.68 → 0.08/0.08/0.13/0.28 (~6x down).
