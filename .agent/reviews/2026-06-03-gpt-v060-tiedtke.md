# GPT v0.6.0 Modified-Tiedtke Handoff

## objective

Port WRF modified Tiedtke cumulus (`cu_physics=6`) into the v0.6.0 frozen
physics interface, build an independent WRF savepoint oracle from pristine WRF
source, and gate the implementation against deep, shallow, and non-triggering
column regimes.

## files changed

- `src/gpuwrf/physics/cumulus_tiedtke.py`
  - Added a WRF-indexed single-column modified-Tiedtke transcription.
  - Returns `PhysicsStepResult`, `PhysicsTendency`, empty cumulus carry, `RAINCV`
    as `rainc_acc`, and cumulus diagnostics including T/Q/U/V tendencies.
- `tests/test_tiedtke_cumulus_oracle.py`
  - Added CPU-only savepoint parity gate and proof-report writer.
- `proofs/v060/oracle/.gitignore`

- `proofs/v060/oracle/dump_to_json.py`

  - Added oracle factory compiling unmodified
    `/home/enric/src/wrf_pristine/WRF/phys/module_cu_tiedtke.F` plus WRF

- `proofs/v060/savepoints/tiedtke_case_1.json` through
  `proofs/v060/savepoints/tiedtke_case_5.json`

- `taskset -c 0-3 proofs/v060/oracle/build_and_run.sh`
- `python -m py_compile src/gpuwrf/physics/cumulus_tiedtke.py`
- `PYTHONPATH=src JAX_PLATFORMS=cpu python - <<'PY' ... tiedtke_column residual probe ... PY`
- `PYTHONPATH=src JAX_PLATFORMS=cpu python -m gpuwrf.contracts.physics_interfaces`
- `PYTHONPATH=src JAX_PLATFORMS=cpu python -m py_compile src/gpuwrf/physics/cumulus_tiedtke.py tests/test_tiedtke_cumulus_oracle.py`

Note: one post-proof interface consistency command was mistakenly run once
without `JAX_PLATFORMS=cpu`; it printed CUDA allocation errors before completing.
It was immediately rerun with `JAX_PLATFORMS=cpu` and passed cleanly. No GPU
validation or performance job was launched.

## proof objects produced

- `proofs/v060/tiedtke_savepoint_parity_report.json`
  - Verdict: `PASS`.
  - Regimes: case 1 deep, case 2 shallow, case 3 deep, cases 4/5 non-triggering.
  - Predeclared tolerances:
    - temperature/moisture tendencies: relative `5e-3`, abs floor `1e-10`
    - momentum diagnostics: relative `5e-3`, abs floor `1e-10`
    - `RAINCV`: relative `1e-3`, abs `2e-4`
    - `KTYPE`: exact
  - Worst meaningful residuals:
    - case 2 shallow `RTHCUTEN` max_rel `0.004233579947631926`
    - case 3 deep `RAINCV` max_abs `0.0001768924321493559`, tolerance `0.0005478181838989258`
  - `full_wrf_exe=false`: this is a real unmodified-WRF module oracle, not a full
    `wrf.exe` integration fixture.

## unresolved risks

- The implementation is a CPU Python/JAX single-column transcription returning
  JAX arrays, not a `jit`/`vmap` GPU-resident production kernel. No GPU
  performance or transfer-audit claim is made.
- The oracle is module-level WRF source, not a full `real.exe`/`wrf.exe`
  fixture. The report records `full_wrf_exe=false` explicitly.
- `cu_physics=16` New Tiedtke is interface-compatible in S0 but was not separately
  gated here; this handoff covers modified Tiedtke `cu_physics=6`.
- The frozen Tiedtke interface has no persistent cumulus carry members; this
  implementation returns an empty cumulus carry.
