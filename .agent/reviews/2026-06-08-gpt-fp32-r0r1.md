- Define valid labels: `fp64_default`, `mixed_perturb_fp32`.
- Keep `fp64_default` as the default for every existing constructor/wrapper.
- Carry the label in report text and static JIT/cache identity.
- Fail closed on unknown labels.
- Do not change `DEFAULT_DTYPES`, `force_fp64`, CLI defaults, or production wrappers.

R1 explicit-base plumbing:

- Thread explicit `BaseState` or equivalent base leaves through:
  `small_step_prep_wrf`, `small_step_finish_wrf`, `_acoustic_core_state_from_prep`,
  `_refresh_grid_p_from_finished`, boundary staging, coupled-core reconstruction,
  restart/init assembly, and carry initialization.
- In `mixed_perturb_fp32`, forbid base/reference recovery by fp32
  `total - perturbation` inside timestep loops.
- Preserve current legacy fallback under `fp64_default` until explicit-base parity
  is proven.

R1 proof gates:

- Default `fp64_default` prep/finish bit identity.
- One-step operational carry identity for fp64 default.
- Static dtype/source audit showing no mixed-mode perturbation/base recovery by
  fp32 total subtraction.
- CPU scalar cancellation probe and one-column acoustic recurrence before any GPU
  run.
