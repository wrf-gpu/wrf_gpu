## Ranked findings

### BLOCKERS
**None.** No integrity violation found. Every operational oracle compares JAX fp64 against the UNMODIFIED pristine Fortran (checksums verified against the live `/home/enric/src/wrf_pristine` tree), at predeclared physical tolerances, with recorded deltas far below those tolerances. Every reference-only scheme is classified reference-only in BOTH the proof JSON AND the actual code wiring (`_SCAN_WIRED_OPTIONS` cross-check matched every claim). ### MAJOR
**None.**

### MINOR
- **M-1 (provenance completeness, cumulus)**: `proofs/v013/savepoints/cumulus/` has no committed `*_source_checksums.txt` (unlike WDM5/WSM7/MRF/Goddard which all commit a checksums file). The build scripts copy pristine modules and the oracle JSON records `self_compare:false`, but a committed checksum file would let a reviewer verify provenance without re-running the build. **Remediation**: have the cumulus `*_build_and_run.sh` emit a `cumulus_source_checksums.txt` into the savepoints dir (one-line addition mirroring the WDM5 script) so reference-only cumulus provenance is checksum-auditable on disk. Low severity (reference-only, `kernel_parity:None`, no operational risk).
- **M-2 (tolerance construction, clear-sky)**: in `clearsky_radiation.json` the pass threshold is `tol_Wm2 = allsky_FLOOR_rmse + constant`, i.e. clear-sky agreement must beat the all-sky calibration floor plus a margin. This is defensible (and the clear-sky RMSEs are mostly far below tol), but the tolerance is data-derived rather than fully predeclared, so a future all-sky regression could loosen the bar. **Remediation**: also assert an absolute predeclared clear-sky ceiling (e.g. clear-sky RMSE ≤ a fixed W/m² per flux) independent of the floor, so the gate cannot drift with the all-sky calibration. Low severity (current deltas pass an absolute reading comfortably).
- **M-3 (trunk hygiene, repeat of caveat)**: see the WDM5-revert caveat above — ensure the release is cut from a tip that RETAINS WDM5, not `de14b5c2`.
