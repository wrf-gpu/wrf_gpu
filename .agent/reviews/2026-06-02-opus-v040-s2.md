## Algorithm reproduced (real.exe-faithful, with file:line refs)

- **Passthrough** (bit-exact, 0.0 diff): XLAT/XLONG(+U/V), MAPFAC_M/U/V(+X/Y),
  F, E, SINALPHA, COSALPHA, HGT (from met_em HGT_M = single-source terrain).
- **TSK**: SKINTEMP, then land lapse `-= 0.0065*(ht - SOILHGT)`
  (`adjust_soil_temp_new`:1057). flag_sst==0 -> SST-protect branch is a no-op.
- **TMN**: SOILTEMP, NOAHMP land lapse `-= 0.0065*ht` (module_soil_pre.F:973) is
  the soil-interp deep endpoint; then over-water `tmn:=tsk` + out-of-[170,400]K
  land fallback to tsk (:3266,:3320) for the FINAL tmn.
- **SST**: flag_sst==0 -> `sst = tsk` (oracle confirms SST==TSK everywhere).
- **XLAND** from landmask (1=land,2=water); SEAICE/SNOW carried (0 here).
- **SOIL 2->4**: sf_surface_physics=4 -> `init_soil_depth_2` (ZS=[0.05,0.25,0.70,
  1.50], DZS=[0.1,0.3,0.6,1.0]) + `init_soil_2_real`. FLAG_SOIL_LAYERS=1 input
  (2 layers, SOIL_LAYERS thicknesses deep-first). Level midpoints -> [5,25] cm;
  have-depths zhave=[0,0.05,0.25,3.0] m; temp column [tsk,st_top,st_deep,tmn] with
  the metgrid layers + tsk + tmn all carrying the over-land elevation lapse;
  moisture column [sm_top,sm_top,sm_deep,sm_deep]; piecewise-linear in depth;
  over-water tslb:=tsk, smois:=1; zero-moisture floor 0.005 (:3379).
- **Categorical (surface_input_source=3 / dominant path)**: IVGTYP=NINT(LU_INDEX),
  LU_INDEX(out)=IVGTYP; ISLTYP=NINT(SCT_DOM) then landmask-match (land+isoilwater
  ->default_soiltype=8; water+non-water->14).

## Honest residuals (NOT masked away)

1. **One static d01 inland-lake cell** (IVGTYP lu=21->5, ISLTYP sct=6->8). The
   oracle wrfinput files are post-`start_em` snapshots, so this cell carries a
   NOAHMP_INIT lake-fill that is NOT derivable from the real.exe-faithful
   dominant-category path (nor from the met_em LANDUSEF at that cell, which has
   only water+lake fractions). It is identical in every d01 case (static geog
   feature). d02/d03 have no lakes and are categorically exact. The gate asserts
   the lake-cell-EXCLUDED categorical comparison (=0 diff); the lake-cell delta is
   recorded per case in the proof. This is a downstream-LSM-init artifact, not an
   S2 real.exe-equivalence defect.
2. **SH2O** is the supercooled liquid/frozen split, computed by NOAHMP_INIT at
   model start — NOT a real.exe field, and absent from `WRFINPUT_TOLS` (out of
   gate). This lane returns `sh2o = smois` (the real.exe pre-LSM-init value;
   water cells 1.0).

## Unresolved risks / notes for the manager (S5 integration)

- **`tmn` two-faced value**: `SurfaceInit.tmn` is the FINAL (water-filled,
  fallback-fixed) deep-soil temp the wrfinput stores. The soil lane needs the
  PRE-fix endpoint, so `soil_init` re-runs the surface lane internally
  (`compute_surface_init_full`) rather than relying on the frozen `SurfaceInit`.
  This is deterministic (same met_em -> same result) and lane-internal; it does
  double the surface compute. If S5 wants to avoid that, the clean fix is an
  additive optional `tmn_soil_endpoint` field on `SurfaceInit` (freeze-compatible
  per S0 policy) — flagged, not applied (types.py is frozen / manager-owned).
- **No `types.py` change needed**; SH2O/vegfra/canwat/snowh/seaice are already
  optional on the handoff types. CANWAT/VEGFRA left None (model-init / monthly-
  interp fields, out of the S2 gate).
- **flag_sst==0 assumed** (loud-rejected if a future met_em carries SST). The
  SST/seaice cold-case branches are documented but not wired (Canary has no ice).
- Surface_input_source=3 (dominant) was confirmed EMPIRICALLY from the oracle
  (ISLTYP==NINT(SCT_DOM) + landmask-match), since the case namelist sets none of
  surface_input_source/default_soiltype/sf_lake_physics (all defaults).
