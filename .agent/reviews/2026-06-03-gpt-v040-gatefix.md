# GPT v0.4.0 Forecast-Gate Harness Fix Handoff

## Objective

Cross-model debug/fix for the Opus-authored v0.4.0 24h native-init forecast-gate
harness. Fix the backfill-output reference-dir bug and stop classifying harness
exceptions / missing h2+ CPU references as numerical BLOWUP. No GPU forecast was
launched.

## Files Changed

- `proofs/v040/s5_forecast_gate_exec.py`
  - Splits CPU-reference output dir from original oracle metadata dir.
  - `build_native_forecast_case(..., metadata_run_dir=...)` now uses the original
    oracle run for `Gen2Run(...).grid()` / namelist and `wrfinput_d01` metrics,
    while scoring still reads wrfout frames from the reference output dir.
  - Adds explicit case classification helpers: `ERROR`, `UNSCOREABLE`,
    `SCORED_PASS`, `SCORED_FAIL`, `BLOWUP`.
- `proofs/v040/run_forecast_gate_24h.py`
  - Threads `oc.run_dir` as metadata source and the resolved ref dir as CPU-output
    source.
  - Emits case-class counts/table and computes final verdict from scored cases
    only.
- `src/gpuwrf/init/real_init/comparator.py`
  - Applies the same metadata/reference split and exception capture to the
    canonical `run_forecast_gate(execute=True)` entrypoint.
- `proofs/v040/gate_harness_fix_audit.py`
  - CPU-only audit/proof generator.
- `proofs/v040/gate_harness_fix_report.json`
  - Proof object.
