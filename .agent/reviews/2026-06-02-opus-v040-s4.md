## Design notes (for reviewers)

- **Masking policy (S0 sec 5)** implemented exactly: TSLB/SMOIS/TMN over land
  (XLAND<1.5), SST over water; categoricals (XLAND/ISLTYP/IVGTYP) exact-on-all;
  non-finite (below-ground/fill) excluded on both sides. PASS = `rmse<=tol AND
  maxabs<=tol`.
- **wrfbdy is COUPLED** (confirmed empirically: `U_BXS` mean ≈ 1.1e6 = field·mu,
  despite the file's `units="m s-1"` attr). The frozen `WRFBDY_TOLS` are scaled
  for coupled values, so the native `LateralBC.values` MUST carry coupled
  quantities (matches the `types.py` LateralBC docstring). Comparator scores the
  coupled VALUE (_BXS/.../_BYE) AND TENDENCY (_BTXS/...), worst-over-4-sides,
  tendency tol = value tol / `bdyfrq_seconds`.
- **Canary wrfbdy has 4 Time frames** — comparator scores the FIRST frame (t0
  value / first-interval tendency). Multi-interval scoring is a future option.

## Unresolved risks / NEEDED `types.py` changes (manager — file is FROZEN)

1. **`ALT` is not a wrfinput variable.** `WRFINPUT_TOLS` has an `ALT` key, but
   real.exe writes only `AL`, `ALB`, `T_INIT`, `P_HYD` (NOT `ALT`) to wrfinput.
   The comparator classifies ALT as `NOT_IN_WRFINPUT` (non-blocking) and checks
   `AL` instead (ALT = AL + ALB is recomputed at runtime). **Recommend the
   manager drop the `ALT` key from `WRFINPUT_TOLS` on the next `types.py` bump**
   (or have the driver emit ALT into a debug wrfinput). Harness-side handled; no
   correctness gap, just a dangling tol key.
2. **wrfbdy moisture coupling:** the oracle wrfbdy also carries QCLOUD/QRAIN/QICE/
   QSNOW/QGRAUP boundaries. The frozen `WRFBDY_TOLS`/`LateralBC.coupled_field_names`
   only specify U/V/T/PH/QV/MU. If S3 produces hydrometeor LBC, the manager should
   add their tol keys + names on a `types.py` bump; the comparator will score any
   field present in BOTH `LateralBC` and the oracle (no code change needed once
   the names are added).
3. **W in wrfbdy:** oracle has `W_BXS` etc.; `WRFBDY_TOLS` has no W key and the
   native W LBC is init-0. Not scored. Fine for v0.4.0 (specified-W LBC unused).
