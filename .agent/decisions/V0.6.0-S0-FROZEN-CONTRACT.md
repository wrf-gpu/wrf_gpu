## 0. What this freeze guarantees

A scheme lane that conforms to this contract is **file-disjoint and
savepoint-gatable in isolation**, and can be merged without touching the dycore,
the wrfout/wrfinput/wrfrst schema, nesting plumbing, or the restart format —
EXCEPT for the three explicitly enumerated append-only State leaves below, whose
materialization is a single manager-owned serial patch.

The freeze does **not** claim any scheme is correct, fast, or forecast-safe.
"Accepted" in the namelist matrix = "interface accepted for lane work."

## 1. The scheme-call payload: `PhysicsTendency` / `PhysicsStepResult`

One scheme call returns a `PhysicsStepResult`:

* `tendency: PhysicsTendency`
  * `state_tendencies` — per-second rates keyed by an allowed **State leaf**
    (`STATE_TENDENCY_KEYS`). This is the WRF `R*TEN` style.
  * `state_replacements` — post-scheme in-place replacement arrays, for schemes
    ported in WRF's "update the array in place" style. A leaf may appear in
    tendencies **or** replacements in one call, never both (`validate_keys`).
  * `accumulator_increments` — per-call mm increments for the precip
    accumulators (`ACCUMULATOR_UPDATE_KEYS`).
  * `diagnostics` — scheme-local diagnostic side channel.
* `carry: PhysicsCarry` — persistent WRF `state:` members that are NOT dycore
  State leaves (see §3).
* `diagnostics: PhysicsDiagnostics` — family-grouped diagnostics (microphysics /
  surface_layer / pbl / cumulus / land_surface / **radiation**).

Allowed `state_tendencies` / `state_replacements` keys (`STATE_TENDENCY_KEYS`):
`u, v, w, theta, qv, qc, qr, qi, qs, qg, Ni, Nr, Ns, Ng, Nc, Nn, qke, ustar,
theta_flux, qv_flux, tau_u, tau_v, rhosfc, fltv, t_skin, soil_moisture, mavail`.
Unknown keys fail `PhysicsTendency.validate_keys()`.

This is the bridge to the runtime `OperationalCarry`; operational dispatch merges
the family subtrees **without any host/device transfer inside the timestep
loop** (project rule).

## 2. The per-scheme adapter spec: `PhysicsStepSpec`

`SCHEME_STEP_SPECS` is the frozen registry of adapter metadata (19 specs: 17
single-option + 2 radiation variants). Each spec carries:

`family, option, name, wrf_slot, owner_module, oracle, reads_state,
writes_state, reads_carry, writes_carry, returns_accumulators, diagnostics,
notes, variant`.

`variant` disambiguates the one case where a single option number maps to two
independent adapters sharing a kernel: **radiation option 4** = RRTMG LW
(`variant="lw"`) + RRTMG SW (`variant="sw"`), since `ra_lw_physics` and
`ra_sw_physics` are distinct namelist switches.

Frozen WRF call slots (`WRF_CALL_ORDER_SLOTS`), in WRF order:

| Slot | WRF source |
| --- | --- |
| `first_rk_pre_radiation_driver` | `dyn_em/module_first_rk_step_part1.F` |
| `first_rk_radiation_driver` | `dyn_em/module_first_rk_step_part1.F` |
| `first_rk_surface_driver` | `dyn_em/module_first_rk_step_part1.F` |
| `first_rk_pbl_driver` | `dyn_em/module_first_rk_step_part1.F` |
| `first_rk_cumulus_driver` | `dyn_em/module_first_rk_step_part1.F` |
| `solve_em_microphysics_driver` | `dyn_em/solve_em.F` |

Each spec's `oracle` MUST be a WRF savepoint gate. No JAX-vs-JAX self-compare is
an oracle.

### Coverage (all 6 categories)

| Family | Options with frozen specs |
| --- | --- |
| microphysics | 1 (Kessler), 6 (WSM6), 8 (Thompson), 10 (Morrison), 16 (WDM6) |
| pbl | 1 (YSU), 5 (MYNN), 7 (ACM2) |
| surface_layer | 1 (sfclayrev), 5 (MYNN-SL), 7 (Pleim-Xiu) |

| land_surface | 2 (Noah classic), 4 (Noah-MP) |
| radiation | 4-lw (RRTMG LW), 4-sw (RRTMG SW) |

Radiation is a clean column endpoint: each adapter writes a **held-rate
`theta` tendency** (WRF `RTHRATEN`, the rate held constant across the `radt`
interval) plus surface radiation diagnostics (`GLW`, `SWDOWN`, `GSW`, …) consumed
by the LSM. It adds no prognostic species.

## 3. The species / scalar Registry — absorbing NEW prognostic species

`physics_registry.py` mirrors the relevant `Registry.EM_COMMON` package
memberships. The design intent: a new scheme declares its species in the registry
maps, and IO / init / nesting consume those maps instead of hardcoding the
Thompson-era set.

### Append-only State rule (THE invariant)

`State.__slots__` order is **frozen prefix-stable**. The only v0.6.0 additive
dycore leaves are appended at the END:

| Append # | Leaf | Registry member | wrfout | dtype gate | Owner lane |
| --- | --- | --- | --- | --- | --- |
| 1 | `Nc` | `qnc` | `QNCLOUD` | FP32-gated | WDM6 / cloud-droplet number |
| 2 | `Nn` | `qnn` | `QNCCN` | FP32-gated | WDM6 / CCN number |
| 3 | `rainc_acc` | `RAINC` | `RAINC` | FP64 | cumulus accumulator |

These are ALREADY materialized in `State.__slots__` (with `None`→zeros defaults
so a pre-v0.6.0 flatten with the old leaf count still reconstructs).

**Generality verdict for new species:** the registry maps
(`MP_MOIST_MEMBERS`, `MP_NUMBER_MEMBERS`, `NUMBER_REGISTRY_MEMBER`,
`NUMBER_WRFOUT_NAME`) are **already general for the v0.6.0 menu** (Kessler →
WDM6, ≤6 moist + ≤4 number species, all on the fixed `(z,y,x)` mass grid). They
are **NOT open-ended**: a scheme that introduces a species not in
`NUMBER_SPECIES`/`MOIST_SPECIES` (e.g. hail `qh`, rime mass/volume `qir/qib`,
multiple ice categories `qi2/qi3`, aerosol `qnwfa/qnifa/qnbca`, 3rd-moment
reflectivity `qzi`) requires a NEW append-only State leaf + the full serial
plumbing patch (slots, precision matrix, restart schema bump, wrfout sourcing,
nest allocation). That patch is the cost gate for every STATE-EXPANDING scheme
in Part B.

### Carry vs State vs Diagnostics — the three storage tiers

| Tier | Storage | Persisted across steps? | wrfout/restart? | Examples |
| --- | --- | --- | --- | --- |
| **State leaf** | `State.__slots__` | yes (scan carry) | yes | moist species, number conc, `qke`, accumulators |
| **PhysicsCarry** | family sibling pytree | yes | restart yes / wrfout optional | KF `w0avg,nca`; Grell-Freitas `cugd_*`; Noah classic `flx4,fvb,fbur,fgsn,smcrel,xlaidyn`; Noah-MP `NoahMPLandState` |
| **PhysicsDiagnostics** | family sibling pytree | no (recomputed) | wrfout optional | `re_cloud/re_ice/re_snow`, `pblh`, `tke_pbl`, `raincv`, radiation surface fluxes |

Rule: WRF `state:` members that are NOT dycore-prognostic stay in `PhysicsCarry`
(the existing Noah-MP carry pattern). Do NOT reinterpret an existing 2-D State
leaf as a multi-layer field (e.g. Noah classic's 4-layer soil uses a land carry,
not `State.soil_moisture`).

## 5. wrfout / wrfinput / wrfrst field set

* Moist + number leaves emit their `Registry`-mapped names
  (`MOIST_WRFOUT_NAME`, `NUMBER_WRFOUT_NAME`); `wrfout_names_for_mp(opt)` returns
  the active set per option so the writer is option-driven, not hardcoded.
* The writer (`io/wrfout_writer.py`) already carries forward-compatible optional
  names for Morrison/WDM6/KF additions (`MICROPHYSICS_EXTRA_VARIABLES`).
* Restart: every State leaf with `restart_required=True` in `FIELD_SPECS` must
  round-trip; the three additive leaves bump the restart schema deliberately.

**Generality verdict:** the wrfout/restart path is **already general** for the
v0.6.0 menu (option-driven name resolution). A new species name not in the
`*_WRFOUT_NAME` maps needs a one-line map entry + restart schema bump (part of
the serial append patch).

## 6. Nesting-aware field list — `nest_field_list(...)`

The single frozen source both v0.5.0 nesting and v0.6.0 physics expansion
consume. Core list explicitly includes `qv` (WRF nests force water vapor even
with no MP). Active MP hydrometeors = forcedown + feedback + lateral-bdy. Active
number/scalar fields and MYNN `qke` = forcedown + feedback; lateral scalar bdy
stays `False` until a lane explicitly implements WRF `have_bcs_scalar`. Callers
must pass the live namelist options, never hardcode `{qv}`.

**Generality verdict:** **already general** — driven by `moist_species_for_mp` /
`number_species_for_mp`. New species ride the same forcedown/feedback machinery
once registered.

## 8. Already-general vs needs-extension — summary

| Interface surface | Already general for the v0.6.0 menu | Needs extension for which schemes |
| --- | --- | --- |
| `PhysicsTendency` / `PhysicsStepResult` | YES (tendency + replacement + accumulator + diag) | none in menu; non-`theta`/non-flux heating channels (e.g. urban canyon temps) would need new keys |
| `PhysicsCarry` family tree | YES | a scheme with a *huge* internal state (RUC 9-layer soil, CLM PFT mosaics, SBM bins) needs a new typed carry subtree (mechanically fine, large) |
| Number/moist species Registry | YES for ≤6 moist + {Ni,Nr,Ns,Ng,Nc,Nn} | hail `qh`, multi-ice, rime `qir/qib/qvolg`, aerosol `qnwfa/qnifa/qnbca`, 3-moment `qzi` → new append-only State leaf + serial patch |
| Accumulators | YES | snow-vs-graupel partition edge cases per scheme |
| wrfout / restart | YES (option-driven names) | new species names → map entry + restart bump |
| `nest_field_list` | YES | scheme-specific lateral scalar BC (`have_bcs_scalar`) if a 2-moment lane needs it |
| WRF call slots | YES (6 frozen slots) | shallow-cumulus (`shcu_physics`) is a 7th driver slot not yet modeled; chem/aerosol feedback slots absent |
| `State.__slots__` | append-only, prefix-frozen | every STATE-EXPANDING scheme = one serial manager patch |

The interface is **general enough that every CLEAN COLUMN ENDPOINT in Part B
plugs in with zero serial plumbing**, and every STATE-EXPANDING scheme costs
exactly one well-defined serial append patch. ARCH-FORCING schemes (bins,
adaptive sub-stepping, data-dependent iteration) are the only ones that stress
the fixed-shape-JAX assumption itself.
