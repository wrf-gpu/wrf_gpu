Freeze the v0.6.0 physics expansion interfaces before per-scheme lanes run in
parallel. This S0 does **not** implement new physics kernels and makes no GPU
performance claim. It freezes the shared Registry/State/wrfout/namelist/nesting
surface so later lanes can be file-disjoint and WRF-savepoint gated before any
integrated forecast.

## Physics Adapter Interface

`PhysicsTendency` is the only scheme update payload:

* `state_tendencies`: per-second rates keyed by allowed State leaves.
* `state_replacements`: WRF in-place style post-scheme replacement arrays.
* `accumulator_increments`: per-call mm increments for precipitation
  accumulators.
* `diagnostics`: scheme-local diagnostic side channel.

Adapters must never return both a tendency and replacement for the same leaf in
one call. Unknown State/accumulator keys fail validation.

`PhysicsCarry` groups persistent non-State WRF `state:` members by family:
`microphysics`, `surface_layer`, `pbl`, `cumulus`, `land_surface`, `radiation`.
This is the bridge to current `OperationalCarry`; operational dispatch must
merge these family subtrees without host/device transfers inside timestep loops.

  tendency namespace.

* Append `Nc,Nn,rainc_acc` to State/precision/restart/writer/nesting allocation.
* Operational dispatcher selection and WRF-slot ordering.
* Nested boundary allocation for active moist/scalar fields.
