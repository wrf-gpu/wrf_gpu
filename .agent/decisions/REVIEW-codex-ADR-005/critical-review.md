# Critical Review - ADR-005 First Physics Suite

Decision: Accept with required fixes

## Top Three Structural Concerns

1. The Thompson recommendation is defensible as a sequencing choice, but the ADR's strongest argument is implementation/validation tractability, not highest Canary operational impact. The scout explicitly ranks PBL/surface layer as the highest meteorological criticality for the Canary regime (`.agent/sprints/2026-05-20-m5-s0-physics-scheme-selection/scout-report.md:21-25`), while the constitution says Canary operational value comes before broad WRF scope (`PROJECT_CONSTITUTION.md:13`). The ADR must be explicit that Thompson-first is a first real-physics stop/go package, not a claim that precipitation microphysics is the highest-value missing operational component.

2. The selected target is under-specified. "Thompson 2008 microphysics using WRF v4.7.1 `mp_physics=8` semantics" is specific enough as a scheme name, but "a constrained WRF-compatible Thompson column subset first" leaves the M5-S1 worker room to implement a toy-like subset and still claim progress (`.agent/decisions/ADR-005-first-physics-suite.md:13`). The contract requires a scheme specific enough that M5-S1 can implement it and validate a Tier 1 fixture (`.agent/sprints/2026-05-20-m5-s0-physics-scheme-selection/sprint-contract.md:87-90`).

3. The JAX implementability case is appropriately cautious in places, but still leans too heavily on the M2 analytic-column surrogate. ADR-001 itself warns that real Thompson/MYNN physics is the unexercised M5 risk (`.agent/decisions/ADR-001-backend-selection.md:79-80`), and its profile caveat labels M2 metrics fallback-profiled and micro-fixture-limited (`.agent/decisions/ADR-001-backend-selection.md:22`). ADR-005 should treat launch/register/local-memory numbers as hypotheses only until a real Thompson artifact exists.

## Findings

1. **Blocker - Freeze the minimum Thompson fixture and implementation boundary before M5-S1.** ADR-005 defers the exact WRF call site to the manager (`.agent/decisions/ADR-005-first-physics-suite.md:35`), admits sedimentation/number-concentration handling may force narrowing (`.agent/decisions/ADR-005-first-physics-suite.md:49`), and only revisits if the result no longer represents `mp_physics=8` semantics (`.agent/decisions/ADR-005-first-physics-suite.md:53`). That is too late. Required fix: add a minimum target surface before dispatch: WRF routine/call-site candidate, included hydrometeor species, whether sedimentation is in or out, number-concentration fields, exact before/after state or tendency outputs, and the Tier 1 fixture variables/tolerances. Without this, the "selected scheme" can collapse into a non-representative subset.

2. **Major - The operational rationale needs a falsifiable sequencing claim.** The scout and ADR both concede PBL is arguably the most important Canary category (`.agent/sprints/2026-05-20-m5-s0-physics-scheme-selection/scout-report.md:23`; `.agent/decisions/ADR-005-first-physics-suite.md:21`), while the project plan's M5 sequence already names microphysics, then PBL, then radiation (`PROJECT_PLAN.md:128`). Required fix: state explicitly that Thompson-first is chosen to prove the WRF fixture/JAX/profile path on a meaningful precipitation package, and that M5/M6 must not describe it as operational sufficiency. Add a follow-on decision hook for MYNN/PBL so the trade-wind inversion and low-cloud risk does not disappear behind a successful precipitation kernel.

3. **Major - Do not use M2 profile numbers as more than a risk prior.** The JAX column artifact reports 1 launch, 0 local memory, and 22 registers/thread (`artifacts/m2/jax/column_profile.json:27-31`), but it also records fallback-derived profiling and profiler limitations (`artifacts/m2/jax/column_profile.json:30`) and is an analytic thermo column, not Thompson. ADR-005's 6-9 launches and 96-128 register estimate (`.agent/decisions/ADR-005-first-physics-suite.md:27`) should be labeled as a dry-run hypothesis, not readiness evidence. Required fix: make the first real Thompson profile JSON the proof object that decides readiness, consistent with ADR-001's M5 stop/go gate (`.agent/decisions/ADR-001-backend-selection.md:131-139`).

4. **Major - The gray-zone gate is too discretionary.** ADR-005 says GO is `<=128` registers, fallback triggers at `>200`, and 129-200 registers require "manager judgment" (`.agent/decisions/ADR-005-first-physics-suite.md:39`). That range is exactly where bad physics kernels can limp forward with low occupancy and no clear accountability. Required fix: define the gray-zone outcome: at minimum one JAX restructuring attempt, documented occupancy/local-memory evidence, and reviewer signoff before proceeding without a fallback mini-ADR.

5. **Minor - Tie the decision to the actual project target stack, not only generic WRF practice.** PROJECT_PLAN records the Gen2 target stack as Thompson, MYNN-EDMF, Noah-MP, and RRTMG (`PROJECT_PLAN.md:176`). ADR-005's evidence pointers mention external-prior training knowledge and local ADRs (`.agent/decisions/ADR-005-first-physics-suite.md:61`) but should cite the local target-stack decision directly so the choice reads as project-specific, not merely "common WRF stack" reasoning.

## Counter-Proposals

I would not reject Thompson outright. It is the best first package if the manager's real objective is "prove a real branchy WRF column scheme under JAX with a WRF savepoint and M5 profile gate." The required fixes above are enough to make that decision auditable.

If I were optimizing strictly for Canary operational impact rather than first-kernel tractability, I would pick **MYNN2.5 / MYNN-EDMF PBL** first. Operational reason: the Canary regime is dominated by marine trade winds, inversion structure, low cloud, near-surface wind, humidity, and terrain-accelerated flow; the scout itself ranks PBL/surface layer highest (`.agent/sprints/2026-05-20-m5-s0-physics-scheme-selection/scout-report.md:23`). JAX implementability case: a constrained MYNN vertical-column target can be staged around stability functions, mixing length, TKE tendency, and a batched vertical diffusion solve using `jax.lax.scan`/bounded loops. It may be more structurally predictable than Thompson's many thresholded hydrometeor branches, though it carries harder coupling to surface fluxes and validation boundaries. That coupling cost is why I accept Thompson-first as a sequencing choice rather than recommending rejection.

A lower-risk microphysics counter-proposal is **WSM6**. Operational reason: it would still exercise explicit precipitation and hydrometeor phase for 3 km showers. JAX implementability case: fewer prognostic complexities and probably lower register pressure than Thompson. I would not choose it unless Thompson's fixture boundary proves unworkable, because it moves away from the target operational stack and gives weaker evidence for the actual scheme the project intends to run.

## Closing Recommendation

Accept the Thompson-first direction only after the ADR is tightened. The manager should patch ADR-005 before M5-S1 dispatch with a frozen minimum Thompson call boundary, an explicit "sequencing not operational sufficiency" statement, and a non-discretionary gray-zone profile rule. With those fixes, Thompson is a fair first M5 scheme and a useful stress test of ADR-001's JAX decision.
